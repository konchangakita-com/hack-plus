# ファクトチェックリスト

記事公開前に必ず確認する項目リスト。誤情報を防ぎ、信頼性を担保する。

## 🔴 Critical（必須確認項目）

### 日付・年号
- [ ] 記事タイトルの年号（例：「2026年版」）
- [ ] 記事内の日付表記
- [ ] 「最新」「現在」などの時制表現が正確か

### 固有名詞
- [ ] 製品名のスペル（ChatGPT / Claude / Cursor / Gemini など）
- [ ] 企業名のスペル（OpenAI / Anthropic / Google など）
- [ ] サービス名の大文字小文字（GitHub / npm / Next.js など）

### 数値情報
- [ ] 価格（ドル表記・円換算）
- [ ] バージョン番号
- [ ] プラン名（Free / Pro / Enterprise など）
- [ ] 制限値（APIコール数、トークン数など）

## 🟡 High（高優先度）

### 技術仕様
- [ ] APIのエンドポイント
- [ ] 対応言語・フレームワーク
- [ ] システム要件
- [ ] ライセンス情報

### 引用・出典
- [ ] 公式ドキュメントへのリンク
- [ ] 引用元の明記
- [ ] リンク切れチェック

### 比較表
- [ ] 各項目の最新情報
- [ ] 比較軸の統一性
- [ ] 空欄・不明項目の明示

## 🟢 Medium（確認推奨）

### 表現の正確性
- [ ] 「無料」と「一部無料」の区別
- [ ] 「対応」と「実験的サポート」の区別
- [ ] 断定表現の根拠確認

### コード例
- [ ] サンプルコードの動作確認
- [ ] コマンドの実行確認
- [ ] 環境依存の明記

## チェックフロー

### Step 1: 執筆時
- 情報源を明記しながら執筆
- 不確実な情報に [要確認] フラグ

### Step 2: 初回レビュー
- Critical 項目を全て確認
- 公式ドキュメントと突合

### Step 3: 最終レビュー（公開直前）
- High 項目を確認
- リンク切れチェック
- 日付が古くなっていないか再確認

## 情報源の信頼性ランク

### A級（最優先）
- 公式ドキュメント
- 公式ブログ
- プレスリリース

### B級（参考）
- 大手メディアの記事
- 技術ブログ（著名エンジニア）
- GitHub Issues/Discussions

### C級（慎重に扱う）
- 個人ブログ
- Reddit/X（旧Twitter）投稿
- 翻訳された二次情報

## よくある誤り（要注意）

### 価格情報
- ❌ 古い価格表を参照
- ❌ 円換算時のレート記載漏れ
- ❌ 税抜/税込の記載漏れ

### バージョン情報
- ❌ Beta版を正式版と誤認
- ❌ 地域限定機能を一般機能と誤認

### 日付表現
- ❌ 「現在」「最新」の時点が不明確
- ❌ 記事執筆日と公開日のズレ
